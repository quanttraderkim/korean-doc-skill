# rubric

이 점수표는 감상평을 대신하는 것이 아니라, `Skill Off / Skill On` 비교를 같은 축에서 남기기 위한 최소 기준입니다.

## 점수 방식

- `0`: 문제 큼 / 문서 목적에 맞지 않음
- `1`: 부분 충족 / 읽히지만 아쉬움 큼
- `2`: 충분히 충족 / 실무 사용 가능

## 평가 축

| 항목 | 질문 | 점수 |
| --- | --- | --- |
| 독자·목적 적합성 | 독자와 문서의 실제 용도(판단·분석·정의·실행·추적·참고)에 맞는 내용과 구조인가 | 0~2 |
| 첫 화면 | 판단 문서는 결론·요청, 분석 문서는 주요 결과, 실행 문서는 시작 조건·절차, 참고 문서는 범위·탐색 기준처럼 유형에 맞는 진입점이 보이는가 | 0~2 |
| 문체 | 공손체·기계적 `~함`·긴 `~한다` 설명문 없이, 연결 논리·병렬 항목·순서가 내용에 맞는 형태로 읽히는가 | 0~2 |
| 용어 | 문맥을 무시한 직역과 자작 약어 없이 실제 업무에서 쓰는 표현으로 이해되며, DAU·API·필드명 같은 통용어는 정확히 유지되는가 | 0~2 |
| 분량 / 절제 | 반복·빈 섹션·불필요한 설명이 없고, 추가된 길이가 정확성·안전성·탐색성 개선에 쓰였는가 | 0~2 |
| 구조 | 제목이 섹션 역할에 맞고, 같은 내용이 반복되지 않으며, 분석 결과는 방법·한계보다 먼저 보이고 PRD·런북·참고 문서의 조회 구조는 보존됐는가 | 0~2 |
| 표 / 불릿 | 표와 불릿이 필요한 곳에만 쓰이고, 실제 열은 있지만 결정에 도움이 없는 표나 펼치기 속 표 과사용이 없는가 | 0~2 |
| Confluence 포맷 | inline code, underscore, 표/불릿 변환이 Confluence에서 깨지지 않는가 | 0~2 |
| 문서 타입 적합성 | 전략·분석·PRD·런북·위클리·참고 위키 등 해당 타입의 사용 방식에 맞고 다른 타입의 형식을 강제하지 않았는가 | 0~2 |
| 가정 / 비용 / 법무 | 검증 전 숫자, 원가, 법무/개인정보보호 가능성을 단정하지 않고 확인 필요로 남겼는가 | 0~2 |

총점은 `20점 만점` 기준으로 보되, 총점보다 어떤 항목이 반복적으로 낮은지가 더 중요합니다.

용어 항목은 아래처럼 판정합니다.

- `0`: 직역투·자작 약어 때문에 영어 원문이나 별도 설명 없이는 뜻을 알기 어려움
- `1`: 대체로 이해되지만 정확한 필드명·공식 명칭이 아닌 일반 문장에 `정본`, `경계값`, `보안 경계`, `오너`, `드라이버`처럼 업무 문맥에 맞지 않는 표현이 남음
- `2`: 문장에서 하는 역할에 맞는 자연스러운 표현을 쓰고 통용되는 전문 용어와 식별자는 정확히 유지

## 해석 기준

- `+2 이상`: 스킬 효과 분명
- `0 ~ +1`: 일부 개선, 규칙 보강 여지 큼
- `음수`: 스킬 또는 프리셋 적용 실패 가능성 높음

점수만 기록하고 끝내지 말고, 낮은 항목은 반드시 [`failure-taxonomy.md`](./failure-taxonomy.md) 코드와 연결하는 것을 권장합니다.
